Location & Motion

iOS

Location

CLLocationManager
  • distanceFilter

    The minimum distance (in meters), moved horizontally, to generate a location update. Can vary from CLDistanceFilterNone for all movements, to very large distances.

  • desiredAccuracy

    No guarantees, but changes how firm of a lock the device tries to acquire:

    • kCLLocationAccuracyBestForNavigation - highest possible, uses all sensors
    • kCLLocationAccuracyBest - best available using GPS (default)
    • kCLLocationAccuracyNearestTenMeters
    • kCLLocationAccuracyHundredMeters
    • kCLLocationAccuracyKilometer
    • kCLLocationAccuracyThreeKilometers
  • activityType

    Setting this allows iOS to pause tracking when the user stops doing the stated activity.

    • fitness - Walking, running, cycling, etc.
    • automotiveNavigation
    • otherNavigation
  • allowDeferredLocationUpdates(distance: , timeout: )

    Allows iOS to store location updates while the app is asleep and deliver them on wake or when either the distance or timeout condition is met.

  • startUpdatingLocation

    Begins location tracking for an active session, based on above state settings

  • allowsBackgroundLocationUpdates

    Enables the app to receive location updates while suspended. Requires a permission popup.

Android

Location

FusedLocationProviderClient
  • LocationRequest

    Container for configurable parameters to the device's location services.

    • setSmallestDisplacement - distance in meters between location updates
    • setInterval - desired time between location updates (millis)
    • setPriority - allows the following priorities:
      • PRIORITY_BALANCED_POWER_ACCURACY - a block
      • PRIORITY_HIGH_ACCURACY
      • PRIORITY_LOW_POWER
      • PRIORITY_NO_POWER - don't use any battery
    • setMaxWaitTime - setting this to larger durations allows the system to batch its location responses. Claims higher accuracy at the expense of immediacy.
  • requestLocationUpdates

    Begins location tracking for an active session, takes in the parameters above. Can either be configured to generate callbacks in the app while active, or to wake the app with location updates when backgrounded.
